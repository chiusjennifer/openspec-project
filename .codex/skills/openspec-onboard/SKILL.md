帶使用者從 0 到 1 跑完一次 OpenSpec：
探索 → 建立 change → proposal/specs/design/tasks → apply → archive。

## 導覽階段
1. Preflight：確認 CLI 可用。
2. 選題：從 codebase 找小而可完成的任務。
3. Explore：短時間釐清問題與範圍。
4. New：建立 change。
5. Proposal：確認 WHY/WHAT。
6. Specs：建立可測試需求。
7. Design：定義 HOW。
8. Tasks：拆成可執行 checklist。
9. Apply：逐項實作。
10. Archive：封存並回顧。
